# 架构与部署主线说明

## 当前主线

当前项目的主线统一为：

- 小程序前端：微信原生小程序
- 核心后端：微信云开发
- 云函数：部署在微信云开发环境
- 云数据库：使用微信云开发数据库
- 管理后台：优先使用云开发静态托管 + 云函数能力

Node.js 版本暂时保留为：

- 备选实现
- 后续增强版
- 未来迁移方案

不是当前默认运行主线。

---

## 腾讯云服务器是否必须

在当前主线下，腾讯云服务器不是必需依赖。

也就是说，第一阶段你不需要把小程序核心功能部署到腾讯云服务器上。

当前应部署在微信云开发里的内容：

- 小程序前端代码
- 云函数
- 云数据库集合
- 管理后台静态页面

---

## 腾讯云服务器什么时候再用

腾讯云服务器在下面场景再接入：

- 你决定切换到 Node.js API 主线
- 你要部署独立管理后台
- 你要接入更复杂的导入服务、任务队列、定时任务
- 你要做自建数据库、日志中心、对象存储中转
- 你要做更复杂的运营系统或多项目共用后台

---

## 当前开发约束

为了避免双后端继续混用，当前阶段遵循以下规则：

1. 小程序默认走云函数，不默认走 HTTP API
2. Node.js 接口只作为备用，不作为当前上线依赖
3. 管理后台中依赖 Node API 的扩展页面先不纳入当前主线，当前已在后台界面层拦截
4. 后续专题、试卷、导入任务等扩展能力，优先补齐云开发版实现

---

## 本阶段结论

对你现在的业务目标：

- 卖激活码
- 用户激活后刷题
- 后台新增科目
- 批量导入题库
- 科目下按专题/真题/模拟训练

最合适的路径是：

- 先在微信云开发主线上做完 MVP
- 腾讯云服务器先保留，不急着接入核心链路
- 等云开发主线稳定后，再决定是否把增强能力迁到服务器
