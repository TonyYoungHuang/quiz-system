# MVP开发进度与上线总览

更新日期：2026-03-31

## 1. 项目定位

本项目是一个基于微信小程序的题库刷题系统，核心商业模式为：

- 在虚拟商品店铺售卖激活码
- 用户在小程序中输入激活码，开通对应科目题库
- 用户按科目、专题、试卷等方式进行刷题
- 后台支持新增科目、批量导入题库、管理专题/试卷、生成和核销激活码

当前主线架构已经统一为：

- 小程序端：微信云开发
- 后台管理端：云开发 + 静态网页管理页
- 云环境：`cloud1-0g8twq2fde2fa6f0`

Node.js 后端目录仍保留，但当前不是投产主线。

## 2. 当前开发进度

### 已完成模块

#### T01 架构收口与环境统一

- 小程序默认运行主线已统一到微信云开发
- HTTP API 混用逻辑已收口，云函数为默认主通道
- 后台管理页环境已统一到 `cloud1-0g8twq2fde2fa6f0`

#### T02 微信身份与权限体系

- 小程序启动时获取微信 `openId`
- 云端权限判定已优先基于微信身份
- 激活、权限查询、权限校验已接入统一用户身份
- 已兼容旧的本地 `temp_user_*` 用户迁移

#### T03 激活码售卖与核销闭环

- 后台支持批次生成激活码
- 支持按来源、状态筛选激活码
- 支持备注、批次号、核销追踪
- 小程序已可正常完成激活码核销

#### T04 科目管理

- 后台支持新增、编辑、停用、删除科目
- 科目列表可展示状态、更新时间、题量、激活码量、授权用户数
- 小程序首页动态读取科目列表，无需前端硬编码

#### T05 题库批量导入

- 后台支持 CSV 导入
- 支持 JSON 导入
- 已支持单选、多选、判断、填空、简答、开放、论述、计算、材料题扩展结构
- 导入前支持预检
- 导入结果支持分批执行与结果反馈

#### T07 专题训练

- 后台支持专题管理
- 题目可归属到专题
- 小程序支持按专题进入刷题

#### T08 真题试卷

- 后台支持试卷管理
- 题目可归属到试卷
- 小程序支持按试卷进入刷题

### 当前已补强能力

- 首页展示题库总量
- 普通刷题支持随机题序
- 普通刷题支持进度记忆
- 中途退出会结算当前成绩
- 错题自动进入错题本
- 错题做对后自动移出
- 已做过的题不再进入下一轮普通随机题库
- 图片题支持后台补图与前台展示
- 前台支持图片点击预览
- 导入复杂 PDF 题库的本地解析工具已具备

截至当前，已实测通过的关键链路包括：

- 小程序首页能展示已导入科目
- 激活码可成功激活科目
- 激活异常提示已改为正常中文
- 进入科目后可正常加载题目
- 题目可随机刷出
- 做题后可提交

## 5. 当前未纳入本次 MVP 承诺范围

以下功能仍不建议作为今晚对外承诺能力：

- 收藏夹
- 我的页中的答题记录
- 多设备之间同步刷题进度
- 多设备之间同步错题本
- 主观题智能判分

说明：

- 收藏夹页面当前仍是占位页
- 答题记录入口当前仍是占位说明
- 当前刷题进度、错题本、已做题去重基于本地存储实现，适合当前 MVP，但不适合宣传为跨设备同步产品能力

## 6. 已知风险记录

### 可接受风险

- 刷题记录为本地存储版，用户换设备或清缓存后不会同步
- 收藏夹和答题记录还未完成
- 复杂图题目前仍需后台人工补图
- 主观题暂不支持自动判分

### 已处理风险

- 首页题量显示错误已修正
- 题目拉取 100 题上限已修正
- 激活/权限异常乱码提示已修正
- 导入后结果不反馈的问题已修正
- 题库随机出题与进度恢复逻辑已补齐

## 9. 当前结论

当前版本已达到“可投产上线的 MVP 刷题小程序”标准。

可以承接的真实业务闭环是：

- 店铺售卖激活码
- 用户激活科目
- 用户按科目/专题/试卷刷题
- 用户做错题后进入错题本复习
- 后台持续新增科目并导入题库

本次上线建议按 MVP 口径投产，不建议把未完成能力写入对外宣传文案。
