# T02 微信身份与权限体系执行记录

## 本次已完成

1. 小程序启动时会优先获取微信 `openid`
2. 当前用户身份已从本地临时 `userId` 主线切换为真实微信身份主线
3. 新增 `getOpenId` 云函数
4. 激活码核销云函数改为优先使用云端上下文中的 `OPENID`
5. 权限查询云函数改为优先使用云端上下文中的 `OPENID`
6. 权限校验云函数改为优先使用云端上下文中的 `OPENID`
7. 首页、激活页、答题页、我的页面都已等待身份初始化完成再读取权限
8. 对历史同设备临时用户增加了一次性迁移兼容

## 本次效果

- 新用户后续激活和权限绑定将优先绑定到真实微信 `openid`
- 同一微信用户清缓存、换设备后，理论上仍可恢复已绑定权限
- 如果是同一台旧设备上的历史临时用户，首次进入时会尝试把旧 `temp_user_*` 记录迁移到当前 `openid`

你需要在微信开发者工具中新增并上传部署这个云函数：

- `miniprogram/cloudfunctions/getOpenId`

已有但已修改、也需要重新上传部署的云函数：

- `miniprogram/cloudfunctions/activateCode`
- `miniprogram/cloudfunctions/getPermissions`
- `miniprogram/cloudfunctions/checkPermission`

## 验收建议

建议按这个顺序验收：

1. 打开小程序，确认“我的”页面能显示用户 ID，而不是新的随机 `temp_user_*`
2. 用新用户激活一个科目，确认 `user_permissions` 中记录的是微信 `openid`
3. 重新进入小程序，确认已激活科目仍能读到
4. 更换测试设备或清理缓存后重新进入，确认权限仍能恢复
