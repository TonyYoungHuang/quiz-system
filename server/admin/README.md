# 管理后台使用说明

## 访问地址

启动后端服务器后，在浏览器中访问：
```
http://localhost:3000/admin
```

```
admin123
```

**安全提示**：生产环境请务必修改密码（在 `admin/js/app.js` 第 3 行）

## 功能说明

### 1. 数据概览

显示系统整体统计数据：
- 科目总数
- 题目总数
- 激活码总数
- 激活用户数

### 2. 科目管理

**功能**：
- 添加新科目
- 编辑科目信息
- 删除科目（会级联删除相关题目和激活码）
- 切换科目启用/禁用状态

**字段说明**：
- **科目名称**：必填，如"教师资格证-小学"
- **分类**：选填，如"教师资格证"
- **图标**：Emoji表情，如 📚
- **描述**：科目简介
- **排序**：数字越大越靠前

### 3. 题目管理

**功能**：
- 单个添加题目
- 批量导入题目（Excel格式）
- 编辑题目
- 删除题目
- 按科目/题型筛选

**题型说明**：
- **单选题**：只有一个正确答案（A/B/C/D）
- **多选题**：可以有多个正确答案（AB/ABC/ABCD）
- **判断题**：正确答案为A或B（A=正确，B=错误）

**批量导入格式**：

```
题型 | 题目内容 | A选项 | B选项 | C选项 | D选项 | 答案 | 解析
单选 | 下列哪个是正确的？ | 选项A | 选项B | 选项C | 选项D | A | 解析内容...
多选 | 以下哪些是正确的？ | 选项A | 选项B | 选项C | 选项D | AB | 解析内容...
判断 | 地球是圆的吗？ | | | | | A | 解析内容...
```

**格式要求**：
- 每行一道题，用 `|` 分隔
- 判断题只需要填写：题型、题目、答案、解析
- 答案格式：单选填字母（A），多选填组合（ABC）
- 题型关键词：包含"多选"为多选题，包含"判断"为判断题

### 4. 激活码管理

**功能**：
- 批量生成激活码（1-100个）
- 查看激活码列表
- 查看激活码状态（已使用/未使用）
- 删除激活码

**生成流程**：
1. 选择科目
2. 输入生成数量
3. 点击生成
4. 复制激活码（可单个复制或全部复制）

**激活码格式**：
- 8-12位随机字符
- 大写字母 + 数字组合
- 示例：`A1B2C3D4`

### 5. 用户管理

查看已激活的用户信息：
- 用户ID
- 已激活的科目
- 激活时间

## API 接口说明

管理后台使用的 API 接口：

### 科目相关
```
GET    /api/v1/exams           - 获取科目列表
GET    /api/v1/exams/:id       - 获取科目详情
POST   /api/v1/exams/admin     - 创建科目
PUT    /api/v1/exams/admin/:id - 更新科目
DELETE /api/v1/exams/admin/:id - 删除科目
```

### 题目相关
```
GET    /api/v1/admin/questions           - 获取所有题目
POST   /api/v1/admin/questions           - 创建题目
PUT    /api/v1/admin/questions/:id       - 更新题目
DELETE /api/v1/admin/questions/:id       - 删除题目
```

### 激活码相关
```
POST   /api/v1/admin/codes/generate - 生成激活码
GET    /api/v1/admin/codes          - 获取激活码列表
DELETE /api/v1/admin/codes/:id      - 删除激活码
```

## 常见问题

### Q: 忘记管理员密码怎么办？
A: 修改 `admin/js/app.js` 文件第 3 行的 `ADMIN_PASSWORD` 变量值。

### Q: 如何修改默认端口？
A: 修改后端 `.env` 文件中的 `PORT` 变量，或修改 `app.js` 中的端口号。

### Q: 批量导入题目失败怎么办？
A: 检查格式是否正确：
- 每行用 `|` 分隔
- 至少包含 7 列（单选/多选）或 4 列（判断）
- 答案必须是有效选项

### Q: 如何备份数据？
A: 使用 MongoDB 的导出命令：
```bash
mongodump --db=quiz-system --out=./backup
```

## 安全建议

1. **修改默认密码**：生产环境务必修改管理员密码
2. **添加访问限制**：在 `app.js` 中配置 CORS 只允许特定域名
3. **启用HTTPS**：生产环境使用 SSL 证书
4. **定期备份**：定期备份数据库
5. **权限验证**：为管理员接口添加真正的身份验证（JWT、Session等）
